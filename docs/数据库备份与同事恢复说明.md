# 数据库备份与同事恢复说明

## 你这边导出备份

在项目根目录运行：

```powershell
.\backend\scripts\backup_db.cmd
```

或：

```powershell
.\backend\scripts\backup_db.ps1
```

导出成功后，备份文件会生成在：

`backend/backups/`

文件名示例：

`gaokao_planning_20260611_170614.sql`

## 发给同事的内容

- GitHub 仓库地址
- 最新的 `.sql` 备份文件
- `.env.example` 作为环境变量模板

## 同事恢复数据库

同事先安装 PostgreSQL，创建好本地数据库和用户，然后在项目根目录放好自己的 `.env` 文件，再运行：

```powershell
.\backend\scripts\restore_db.cmd D:\path\to\gaokao_planning_xxx.sql
```

或：

```powershell
.\backend\scripts\restore_db.ps1 -BackupFile D:\path\to\gaokao_planning_xxx.sql
```

## 启动项目

先启动后端：

```powershell
.\backend\scripts\start_backend.ps1
```

再启动前端：

```powershell
npm run dev
```

访问地址：

- Frontend: `http://127.0.0.1:5173`
- Backend health: `http://127.0.0.1:8000/api/health`

## 注意事项

- `backend/backups/` 不应提交到 GitHub。
- 这些脚本会优先使用项目内 `.local/postgresql`，如果没有，则自动使用系统 `PATH` 里的 `pg_dump` / `psql`。
- 数据有变化时，重新导出一个新的 `.sql` 文件发给同事。
- 这种方式适合开发、演示和本地联调，不适合多人实时共用同一份业务数据。
