# 高考志愿智能决策辅助系统 / 歪歪志愿馆：项目总目标说明

你现在接手的是一个“高考志愿智能决策辅助系统 / 歪歪志愿馆”项目。

这个系统的目标不是简单做一个查学校、查分数线的小工具，而是要做一个面向高考生和家长的志愿填报辅助决策系统。系统要通过“八字 / 前六字分析 + 高考分数位次 + 历史招生数据 + 专业趋势 + 城市产业 + 家庭偏好”综合生成可解释、可落地的志愿填报建议报告。

系统最核心的逻辑是：

先用八字 / 前六字分析学生适合什么样的发展方向、职业类型、学习方式和人生路径；  
再把八字 / 前六字分析结果映射到适合的专业大类和具体专业方向；  
再结合高考分数、位次、省份、选科、历史录取数据，判断这些专业方向里有哪些院校和专业是学生现实中可以报考的；  
最后根据院校层次、城市产业、就业趋势、家庭偏好，生成冲、稳、保志愿方案和第一志愿建议。

也就是说：

- 八字 / 前六字分析 = 专业方向推荐的核心依据；
- 分数位次 = 判断能不能报、能不能录的现实边界；
- 院校数据 = 判断具体能报哪些学校和专业；
- 城市产业 = 判断未来就业和发展环境；
- 家庭偏好 = 判断方案是否容易被学生和家长接受；
- 性格、兴趣、星座等 = 辅助解释和沟通，不作为核心录取依据。

系统最终逻辑必须体现：

- 八字定方向；
- 分数定位次边界；
- 专业定主线；
- 院校定方案；
- 城市和家庭偏好做修正；
- 最后输出冲稳保和第一志愿建议。

---

## 一、系统最终要实现的核心能力

### 1. 学生信息录入

系统需要支持录入或选择学生基础信息，包括：

- 姓名
- 性别
- 出生时间
- 八字 / 前六字信息
- 省份
- 科类 / 选科
- 模拟分或预估分
- 位次，如果没有位次则根据一分一段表估算
- 兴趣方向
- 性格特点
- 家庭偏好
- 是否接受外省
- 城市偏好
- 专业偏好
- 是否有明确目标专业
- 是否有保研、考研、就业、考公、出国等长期路径偏好

其中，出生时间、八字 / 前六字信息要作为专业方向分析的重要输入。

---

### 2. 八字 / 前六字分析

系统需要先根据学生八字 / 前六字分析出学生适合的发展方向，而不是直接根据分数推荐热门专业。

八字 / 前六字分析需要输出：

- 性格底色
- 学习方式
- 思维类型
- 抗压能力
- 决策风格
- 表达能力
- 逻辑能力
- 稳定性
- 行动力
- 适合的行业气质
- 适合的发展路径
- 不太适合的方向和风险点

分析结果要能归纳为几类发展倾向，例如：

- 技术型
- 研究型
- 稳定型
- 管理型
- 表达型
- 创意型
- 服务型
- 商业型
- 资源整合型
- 规则秩序型
- 实践工程型

---

### 3. 八字结果到专业方向的映射

系统必须实现“八字 / 前六字结果 → 专业大类 → 具体专业”的映射逻辑。

示例映射方向：

#### 技术、逻辑、系统思维、规则推理

如果学生适合技术、逻辑、系统思维、规则推理：

可推荐计算机类、电子信息类、自动化类、数学类、统计类、机械类、电气类、人工智能、数据科学与大数据技术等。

#### 表达、传播、审美、沟通

如果学生适合表达、传播、审美、沟通：

可推荐新闻传播类、汉语言文学、广告学、网络与新媒体、设计类、外语类、数字媒体艺术等。

#### 秩序、规则、稳定、责任

如果学生适合秩序、规则、稳定、责任：

可推荐法学、财会、审计、公安类、公共管理、师范类、档案学等。

#### 照护、耐心、服务、稳定

如果学生适合照护、耐心、服务、稳定：

可推荐医学、护理学、康复治疗学、心理学、教育学、特殊教育等。

#### 商业、资源整合、人际互动

如果学生适合商业、资源整合、人际互动：

可推荐工商管理、市场营销、金融学、经济学、人力资源管理、电子商务、国际经济与贸易等。

#### 研究、深造、理论学习

如果学生适合研究、深造、理论学习：

可推荐数学、物理、化学、生物科学、基础医学、历史学、哲学、社会学等。

#### 动手、工程、实践落地

如果学生适合动手、工程、实践落地：

可推荐土木工程、机械工程、车辆工程、建筑学、电气工程、材料类、能源动力类等。

#### 稳定、教育、长期积累

如果学生适合稳定、教育、长期积累：

可推荐师范类、医学类、财会类、法学类、公共管理类、汉语言文学等。

系统需要有一个 `major_direction_mapper` 或类似模块，专门负责把八字 / 前六字分析结果映射到专业大类和专业关键词。

### 4. 数据资产管理

项目需要逐步整理并使用这些数据目录：

#### 01_各省高考政策

用于记录各省高考模式、批次设置、志愿单位、投档规则、选科要求等。

#### 02_各省一分一段表

#### 03_各省录取分数线和最低位次

- 年份

- 院校名称
- 院校代码
- 院校专业组代码，如果是新高考省份

- 最低分
- 最低位次
- 招生计划数，如有
- 批次
- 备注

#### 04_全国大学基本信息

用于补充院校层次和标签，包括：

- 所在城市
- 院校类型
- 软科 / 校友会 / 官方层级标签，如有

#### 05_本科专业目录

用于专业解释和专业大类匹配，包括：

- 专业代码
- 专业名称

- 学位类型
- 专业简介
- 适合人群
- 就业方向
- 考研方向
- 风险提示

另外还需要导入或维护：

- 专业大类库
- 八字 / 前六字到专业方向的映射库
- 城市产业库
- 样例学生数据
- 报告模板字段
- 志愿推荐规则配置
- 99 元报告模板
- 399 元报告模板

### 5. 专业优先推荐逻辑

系统推荐逻辑不能写成：

> 因为学生分数到了，所以推荐这个专业。

而应该写成：

> 根据学生八字 / 前六字分析，该学生更适合某种发展路径，因此优先推荐某些专业方向。再结合学生当前分数、位次、省份和历史录取数据，在可报考范围内筛选出以下院校和专业。

推荐流程应该是：

1. 八字 / 前六字分析学生适合的职业和发展方向。
2. 把分析结果映射为第一推荐专业大类、第二推荐专业大类、第三推荐专业大类、可接受备选专业、不建议优先选择专业。
3. 在历史录取数据中优先筛选这些专业方向相关的院校专业。
4. 结合学生分数、位次、省份、选科、院校专业组、历史最低分、历史最低位次，判断现实可报范围。
5. 生成冲刺、稳妥、保底志愿。
6. 给出第一志愿建议，并解释为什么这个学校和专业适合作为第一志愿。

---

### 6. 现实录取筛选逻辑

专业方向确定后，再根据：

- 学生省份
- 科类 / 选科
- 分数
- 位次
- 历史最低分
- 历史最低位次
- 院校专业组
- 专业名称
- 招生计划

筛选可以实际填报的学校和专业。

推荐时优先使用最低位次做匹配，分数只作为辅助。

根据历史最低位次与学生位次差距，划分：

- 冲刺志愿
- 稳妥志愿
- 保底志愿

每个推荐结果尽量输出：

- 推荐院校
- 推荐专业
- 所在城市
- 往年最低分
- 往年最低位次
- 与学生位次差距
- 推荐类型：冲 / 稳 / 保
- 推荐理由
- 风险提示

---

### 7. 当八字推荐方向和现实分数冲突时的处理

如果八字 / 前六字分析出来最适合的专业方向，学生当前分数或位次不支持，系统不能硬推。

应该给出替代路径：

- 首选专业方向
- 次选专业方向
- 可接受专业方向
- 不建议优先选择的方向

例如：

> 从八字 / 前六字分析看，学生更适合技术型、系统型专业，首选计算机类和电子信息类。但结合当前位次，优质院校的计算机类竞争较高，因此建议采用：稳妥选择电子信息、自动化、电气类；保底选择信息管理、软件工程中外合作、数据科学相关低位次院校；如果更重视学校层次，可考虑同层次院校中相近专业。

---

### 8. 城市匹配逻辑

系统需要把城市作为志愿决策的重要辅助维度。

需要实现：

- 院校所在城市展示
- 城市产业标签展示
- 城市与专业就业机会匹配
- 本省 / 外省策略分析
- 家庭可接受度提示
- 城市生活成本、就业机会、产业方向的基础说明

但城市不能超过专业主线。城市是辅助修正，不是核心推荐依据。

---

### 9. 学生画像分析

系统可以包含“前六字 / 八字 / 性格 / 星座”等辅助分析，但必须注意合规边界。

要求：

- 八字 / 前六字用于判断适合方向和沟通表达，不用于判断录取概率。
- 前六字、星座、性格分析只能作为辅助解释，不能作为录取依据。
- 报告中必须明确：最终决策必须以官方招生政策、院校招生章程、真实分数位次、当年招生计划和家庭实际情况为准。
- 学生画像主要用于辅助判断：
  - 决策风格
  - 专业偏好
  - 城市适应性
  - 学习压力承受能力
  - 家庭沟通方式
  - 长期发展路径

---

### 10. 报告生成

系统需要支持生成两类报告：

#### A. 99 元基础版报告

重点是快速给出学生的基本定位和初步建议，包括：

- 学生基本信息
- 八字 / 前六字简要分析
- 适合专业方向
- 分数 / 位次定位
- 可选院校范围
- 推荐专业方向
- 冲稳保初步列表
- 城市建议
- 风险提示
- 下一步建议

99 元报告也要体现“八字定方向，分数定边界”，但内容可以简洁。

#### B. 399 元深度版报告

重点是更完整、更像咨询服务交付物，包括：

- 学生八字 / 前六字核心分析
- 学生性格和发展路径分析
- 第一推荐专业大类
- 第二推荐专业大类
- 第三推荐专业大类
- 可接受备选专业方向
- 不建议优先选择的专业方向
- 每个专业方向的推荐理由
- 分数位次分析
- 省内外机会分析
- 院校层次分析
- 专业适配分析
- 城市产业分析
- 冲稳保志愿推荐表
- 第一志愿建议
- 专业优先 / 学校优先 / 城市优先策略
- 家庭沟通建议
- 风险提示
- 最终填报建议
- 后续行动清单

399 元报告中，专业推荐必须放在核心位置。不能让院校推荐压过专业方向分析。

## 二、当前最重要的开发优先级

现在项目的核心问题是：

报告里已经有一些信息，但推荐还不够具体，没有明确告诉用户“根据八字 / 前六字适合什么专业方向、推荐哪个学校、哪个专业、第一志愿填什么”。

因此当前优先级如下：

### 第一优先级

让系统能根据学生八字 / 前六字输出适合的专业方向。

### 第二优先级

建立“八字 / 前六字结果 → 专业大类 → 具体专业名称”的映射规则。

### 第三优先级

让系统能在适合的专业方向内，基于样例学生数据和本地录取数据，输出具体的院校 + 专业推荐表。

### 第四优先级

把推荐结果分成冲、稳、保三类，并解释推荐理由和风险。

### 第五优先级

在 399 元报告中加入“第一志愿建议”和“志愿组合策略”。

### 第六优先级

完善 99 元和 399 元报告的结构差异。

### 第七优先级

再考虑 PDF 导出、美化、页面优化等展示问题。

## 三、推荐结果最低要求

每份报告至少要输出类似这样的结构：

### 1. 学生八字 / 前六字分析

- 核心性格
- 学习方式
- 思维特点
- 适合发展路径
- 不适合方向
- 专业选择提醒

### 2. 适合专业方向

- 第一推荐专业大类
- 第二推荐专业大类
- 第三推荐专业大类
- 可接受备选专业方向
- 不建议优先选择的专业方向

### 3. 学生定位

- 当前分数
- 当前位次
- 所属省份
- 科类 / 选科
- 可冲击院校层次
- 稳妥院校层次
- 保底院校层次

### 4. 冲刺志愿

表格字段：

- 院校名称
- 专业名称
- 城市
- 往年最低分
- 往年最低位次
- 位次差
- 推荐理由
- 风险提示

### 5. 稳妥志愿

字段同上。

### 6. 保底志愿

字段同上。

### 7. 第一志愿建议

必须明确输出：

- 建议第一志愿院校
- 建议第一志愿专业
- 为什么这个专业符合八字 / 前六字分析结果
- 为什么这个学校适合作为第一志愿
- 有什么风险
- 如果用户更重视学校 / 专业 / 城市，分别怎么调整

### 8. 专业方向建议

输出 3 到 5 个适合专业方向，并说明：

- 为什么适合这个学生
- 未来适合做什么
- 学习难点是什么
- 是否需要考研
- 就业稳定性如何
- 家庭是否容易接受

### 9. 城市建议

输出 3 到 5 个适合城市或城市层级。

### 10. 家庭决策建议

帮助家长理解为什么这样填，不要只看学校名气，也要看八字 / 前六字适合方向、专业、城市和未来路径。

---

## 四、重要合规要求

系统必须始终提示：

- 本系统是志愿填报辅助决策工具，不保证录取。
- 所有建议必须以当年官方招生计划、考试院数据、院校招生章程为准。
- 历史录取数据只能作为参考。
- 八字 / 前六字、星座、性格分析只能作为专业方向选择、沟通解释和自我认知辅助工具，不能作为录取依据。
