## 2. 设计定位

本系统属于教育咨询与高考志愿规划类后台系统，界面应体现“专业、可信、清晰、稳重”的设计方向。

### 2.1 设计关键词

| 关键词 | 说明 |
|---|---|
| 专业 | 避免娱乐化、过度卡通化，保持教育咨询系统的正式感 |
| 清晰 | 信息层级明确，方便咨询师快速查看学生情况 |
| 稳重 | 色彩克制，不使用过强刺激性颜色 |
| 可信 | 风险提示、数据说明、免责声明要清楚可见 |
| 高效 | 常用操作入口明显，减少重复点击和长表单压力 |

### 2.2 不推荐的风格

1. 不使用大面积霓虹渐变、强烈赛博风。
2. 不使用娱乐小程序式的夸张图标和动效。
3. 不使用过多红色，避免造成家长焦虑。
4. 不在页面中出现“保证录取”“必上本科”等绝对化表达。
5. 不把所有信息堆在一页，应通过卡片、步骤条和折叠区分层展示。

## 3. 色彩规范

### 3.1 主色系统

| 用途 | 色值 | 使用场景 |
|---|---|---|
| 主色 Primary | `#2563EB` | 主按钮、导航高亮、关键数据强调 |
| 主色浅背景 | `#EFF6FF` | 主色卡片背景、选中项背景 |
| 主色深色 | `#1D4ED8` | 主按钮悬停、重点链接悬停 |
| 页面背景 | `#F8FAFC` | 系统整体背景 |
| 卡片背景 | `#FFFFFF` | 表单、表格、数据卡片 |
| 分割线/边框 | `#E2E8F0` | 表格边框、卡片边框、输入框边框 |

### 3.2 文字颜色

| 用途 | 色值 | 使用场景 |
|---|---|---|
| 一级文字 | `#0F172A` | 页面标题、核心数据、重要说明 |
| 二级文字 | `#334155` | 模块标题、表格主要内容 |
| 三级文字 | `#64748B` | 辅助说明、字段备注、次要信息 |
| 占位文字 | `#94A3B8` | 输入框 placeholder、空状态说明 |
| 反白文字 | `#FFFFFF` | 主按钮文字、深色背景文字 |

### 3.3 功能色

| 类型 | 色值 | 浅背景 | 使用场景 |
|---|---|---|---|
| 成功/安全 | `#22C55E` | `#F0FDF4` | 低风险、已完成、已交付 |
| 提醒/中风险 | `#F59E0B` | `#FFFBEB` | 待补充、中风险、需复核 |
| 危险/高风险 | `#EF4444` | `#FEF2F2` | 高风险、删除、严重错误 |
| 信息/说明 | `#0EA5E9` | `#F0F9FF` | 数据说明、普通提示 |
| 紫色/复核 | `#8B5CF6` | `#F5F3FF` | 已人工复核、AI生成状态 |

### 3.4 色彩使用原则

1. 页面大面积背景使用浅灰白，不使用纯白铺满整个页面。
2. 主按钮统一使用蓝色，不同页面不要随意更换主色。
3. 红色只用于高风险、删除、错误提示，不用于普通强调。
4. 冲稳保颜色建议：冲一冲使用橙色或红色弱化版，稳一稳使用蓝色，保一保使用绿色。
5. 页面中同一类状态标签必须使用同一种颜色，避免造成理解混乱。

## 4. 字体与字号规范

### 4.1 字体族

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;
```

### 4.2 字号规范

| 层级 | 字号 | 字重 | 使用场景 |
|---|---:|---:|---|
| 页面主标题 | 24px | 600/700 | 工作台、学生管理、报告生成等页面标题 |
| 一级模块标题 | 20px | 600 | 大模块标题，如“估分分析”“专业推荐” |
| 二级模块标题 | 18px | 600 | 卡片标题、表单分组标题 |
| 小标题 | 16px | 600 | 表格区域标题、卡片内标题 |
| 正文 | 14px | 400 | 表格内容、字段内容、说明文字 |
| 辅助文字 | 13px | 400 | 表单提示、卡片备注 |
| 标签文字 | 12px | 500 | 状态标签、风险标签 |
| 大数字 | 28px-36px | 700 | 总分、适配度、报告数量等核心数据 |

### 4.3 行高规范

| 文本类型 | 行高 |
|---|---:|
| 标题 | 1.3 |
| 正文 | 1.6 |
| 表格文字 | 1.5 |
| 标签文字 | 1.2 |
| 说明文字 | 1.6 |

### 4.4 字体使用原则

1. 页面标题不要超过一行，必要时使用副标题补充说明。
2. 核心数据可加粗，但不要全部文字都加粗。
3. 辅助说明用灰色，不用黑色，避免视觉负担。
4. 报告预览区正文建议使用 15px 或 16px，提高阅读体验。

## 5. 间距与布局规范

### 5.1 基础间距单位

系统采用 4px 网格体系，常用间距如下：

| 间距 | 数值 | 使用场景 |
|---|---:|---|
| xs | 4px | 图标与文字之间 |
| sm | 8px | 标签间距、小按钮内边距 |
| md | 12px | 表单项内部、卡片小间距 |
| lg | 16px | 卡片内部常规间距 |
| xl | 24px | 模块与模块之间 |
| 2xl | 32px | 页面大区块之间 |
| 3xl | 40px | 页面顶部与主内容间距 |

### 5.2 页面边距

| 区域 | 规范 |
|---|---|
| 主内容区外边距 | 24px |
| 卡片内边距 | 20px / 24px |
| 表格上方筛选区间距 | 16px |
| 页面标题与内容间距 | 20px |
| 模块之间间距 | 24px |
| 表单项上下间距 | 18px-22px |

### 5.3 后台整体布局尺寸

| 区域 | 尺寸 |
|---|---:|
| 顶部栏高度 | 56px / 64px |
| 左侧导航宽度 | 220px |
| 左侧导航收起宽度 | 64px |
| 主内容最小宽度 | 1024px |
| 内容最大宽度 | 可根据页面需要设置为 1440px |

### 5.4 页面布局原则

1. 页面主体采用“顶部标题区 + 内容卡片区”的结构。
2. 筛选区放在表格上方，使用白色卡片承载。
3. 数据概览建议使用 4 列卡片布局。
4. 表单页面不使用单页长表单，必须用步骤条或分组卡片。
5. 结果页优先使用卡片和分栏，不要全部使用长表格。

## 6. 圆角与阴影规范

### 6.1 圆角

| 组件 | 圆角 |
|---|---:|
| 页面卡片 | 12px |
| 按钮 | 8px |
| 输入框 | 8px |
| 标签 | 6px |
| 弹窗 | 12px |
| 数据卡片 | 14px |
| 报告预览卡片 | 12px |

### 6.2 阴影

| 类型 | 样式 | 使用场景 |
|---|---|---|
| 轻阴影 | `0 1px 3px rgba(15, 23, 42, 0.08)` | 普通卡片 |
| 中阴影 | `0 8px 24px rgba(15, 23, 42, 0.08)` | 弹窗、浮层 |
| 悬停阴影 | `0 10px 28px rgba(37, 99, 235, 0.12)` | 可点击卡片 hover |

### 6.3 使用原则

1. 后台系统不建议使用过重阴影。
2. 卡片默认轻阴影或浅边框即可。
3. 弹窗、下拉面板、浮层可以使用中阴影。
4. 表格不使用明显阴影，保持清爽。

## 7. 按钮规范

### 7.1 按钮类型

| 类型 | 样式 | 使用场景 |
|---|---|---|
| 主按钮 | 蓝底白字 | 新增学生、提交、生成报告、保存 |
| 次按钮 | 白底蓝边蓝字 | 返回、取消、查看详情 |
| 普通按钮 | 白底灰边深色字 | 查询、重置、导出 |
| 危险按钮 | 红色文字或红底白字 | 删除学生、删除报告 |
| 文本按钮 | 无背景，仅文字 | 表格操作栏中的查看、编辑 |

### 7.2 按钮尺寸

| 尺寸 | 高度 | 水平内边距 | 使用场景 |
|---|---:|---:|---|
| 大按钮 | 40px | 20px | 表单提交、登录按钮 |
| 默认按钮 | 36px | 16px | 页面常规操作 |
| 小按钮 | 32px | 12px | 表格操作、紧凑区域 |
| 图标按钮 | 32px | 8px | 搜索、更多操作 |

### 7.3 按钮文字规范

| 场景 | 推荐文案 |
|---|---|
| 新建学生 | 新增学生 |
| 保存表单 | 保存 |
| 提交表单 | 提交信息 |
| 进入下一步 | 下一步 |
| 返回上一步 | 上一步 |
| 生成报告 | 生成报告 |
| 导出文件 | 导出 PDF / 导出 Word |
| 删除数据 | 删除 |
| 危险确认 | 确认删除 |

### 7.4 按钮使用原则

1. 每个页面只保留一个最重要的主按钮。
2. 危险操作不能使用主色按钮，必须使用红色或二次确认弹窗。
3. 表格操作栏按钮不宜超过 4 个，超过时使用“更多”下拉菜单。
4. 表单底部按钮顺序建议：左侧“上一步/取消”，右侧“下一步/保存/提交”。

## 8. 卡片规范

### 8.1 基础卡片样式

| 属性 | 规范 |
|---|---|
| 背景色 | `#FFFFFF` |
| 圆角 | 12px |
| 内边距 | 20px / 24px |
| 边框 | `1px solid #E2E8F0` |
| 阴影 | 轻阴影，可选 |

### 8.2 数据概览卡片

用于工作台、估分分析、报告管理等页面。

内容结构：

```text
卡片标题
核心数字
变化说明/辅助说明
```

示例：

```text
今日新增学生
12
较昨日增加 3 人
```

### 8.3 学生信息卡片

用于学生详情页、估分分析页顶部。

建议展示：

| 字段 | 示例 |
|---|---|
| 姓名 | 张三 |
| 省份 | 河南 |
| 高考年份 | 2026 |
| 选科 | 物理+化学+生物 |
| 估分 | 485 |
| 当前状态 | 已完成估分 |

### 8.4 专业推荐卡片

内容结构：

```text
专业名称 + 适配度标签
专业类型
推荐理由
学习难度 / 就业稳定性 / 是否建议考研
风险提示
```

### 8.5 院校推荐卡片

内容结构：

```text
学校名称
城市 / 办学性质 / 学校层次
推荐专业
往年最低分 / 往年最低位次
风险等级
推荐理由
```

### 8.6 卡片使用原则

1. 卡片标题位于左上角，操作按钮位于右上角。
2. 同一页面内卡片高度尽量统一。
3. 可点击卡片需要有 hover 效果。
4. 普通展示卡片不需要明显动效。
5. 风险提示类卡片可以加浅色背景，但不要使用大面积纯红。

## 9. 表格规范

### 9.1 表格基础样式

| 属性 | 规范 |
|---|---|
| 表头背景 | `#F8FAFC` |
| 表头文字 | `#334155`，14px，600 |
| 表格正文 | `#334155`，14px，400 |
| 行高 | 48px / 52px |
| 边框 | 仅保留横向分割线 |
| hover 背景 | `#F8FAFC` |

### 9.2 学生管理表格字段

| 字段 | 宽度建议 | 说明 |
|---|---:|---|
| 姓名 | 120px | 可点击进入详情 |
| 省份 | 100px | 高考报名省份 |
| 高考年份 | 100px | 如 2026 |
| 选科 | 160px | 如物化生 |
| 估分 | 100px | 右对齐 |
| 状态 | 140px | 使用状态标签 |
| 更新时间 | 160px | YYYY-MM-DD HH:mm |
| 操作 | 220px | 查看、编辑、生成报告、更多 |

### 9.3 表格操作栏规范

常用操作顺序：

```text
查看 | 编辑 | 生成报告 | 更多
```

危险操作如“删除”放入“更多”中，且使用红色文字。

### 9.4 表格空状态

当表格无数据时显示：

```text
暂无学生数据，请点击“新增学生”开始录入。
```

### 9.5 表格使用原则

1. 表格适合管理列表，不适合展示长篇分析结果。
2. 数字字段右对齐，文字字段左对齐。
3. 状态、风险、报告类型等字段必须使用标签展示。
4. 表格默认分页，每页建议 10 条或 20 条。
5. 表格上方筛选区不应超过两行，复杂筛选可折叠。

| 状态 | 文字颜色 | 背景色 | 使用场景 |

| 风险 | 文字颜色 | 背景色 | 文案 |
|---|---|---|---|
| 高风险 | `#EF4444` | `#FEF2F2` | 高风险 |

| 类型 | 建议颜色 | 说明 |
|---|---|---|
| 冲一冲 | 橙色/红色弱化 | 可尝试，但风险较高 |
| 稳一稳 | 蓝色 | 与当前定位较匹配 |
| 保一保 | 绿色 | 相对安全，防止滑档 |

### 10.4 标签样式

```css
.tag {
  display: inline-flex;
  align-items: center;
  height: 24px;
  padding: 0 8px;
  border-radius: 6px;
  font-size: 12px;
  font-weight: 500;

```

### 10.5 标签使用原则

1. 标签文字不超过 6 个汉字。
2. 同一含义在所有页面必须使用同一颜色。
3. 标签不要过度堆叠，单个字段最多展示 2-3 个标签。
4. 高风险标签旁边应提供说明或提示图标。

## 11. 表单规范

### 11.1 表单布局

学生信息录入页采用步骤条表单：

```text
基础信息 → 成绩信息 → 报考偏好 → 兴趣画像 → 确认提交
```

### 11.2 表单列数

| 场景 | 推荐列数 |
|---|---:|
| 登录页 | 1列 |
| 学生基础信息 | 2列 |
| 成绩信息 | 3列或2列 |
| 报考偏好 | 2列 |
| 兴趣画像 | 2列 |
| 确认页 | 分组展示 |

### 11.3 表单组件尺寸

| 组件 | 高度 |
|---|---:|
| 输入框 | 36px / 40px |
| 下拉框 | 36px / 40px |
| 日期选择器 | 36px / 40px |
| 文本域 | 最小 96px |
| 单选框组 | 32px-40px |
| 评分组件 | 32px |

### 11.4 必填与提示

1. 必填字段使用红色星号。
2. 复杂字段下方增加说明文案。
3. 成绩字段必须限制输入范围。
4. 输入错误时，错误提示显示在字段下方。
5. 表单提交失败时，滚动定位到第一个错误字段。

### 11.5 表单错误文案

| 场景 | 文案 |
|---|---|
| 姓名为空 | 请输入学生姓名 |
| 省份为空 | 请选择高考报名省份 |
| 分数超出范围 | 分数超出合理范围，请重新填写 |
| 出生日期错误 | 出生日期不能晚于当前日期 |
| 选科为空 | 请选择学生选科组合 |
| 家庭预算为空 | 请填写或选择家庭预算 |

## 12. 步骤条规范

### 12.1 使用场景

步骤条主要用于新增学生和编辑学生资料页面，避免长表单造成填写压力。

### 12.2 步骤顺序

```text
基础信息 → 成绩信息 → 报考偏好 → 兴趣画像 → 确认提交
```

### 12.3 状态样式

| 状态 | 样式 |
|---|---|
| 已完成 | 蓝色圆点 + 对勾 |
| 当前步骤 | 蓝色高亮 |
| 未完成 | 灰色圆点 |
| 错误步骤 | 红色提示 |

### 12.4 交互原则

1. 用户可返回上一步修改。
2. 未完成当前步骤时，禁止进入下一步。
3. 切换步骤时保留已填写数据。
4. 离开页面前如有未保存内容，需要弹出确认提示。

## 13. 弹窗与提示规范

### 13.1 弹窗类型

| 类型 | 使用场景 |
|---|---|
| 确认弹窗 | 删除学生、生成报告、导出报告 |
| 表单弹窗 | 简单编辑、备注修改 |
| 详情弹窗 | 查看专业说明、院校详情 |
| 警告弹窗 | 缺少关键信息、存在高风险 |

### 13.2 常用确认文案

| 场景 | 标题 | 内容 |
|---|---|---|
| 删除学生 | 确认删除学生？ | 删除后无法恢复，请确认是否继续。 |
| 生成报告 | 确认生成报告？ | 报告生成后仍需人工复核，不代表最终录取结果。 |
| 导出报告 | 导出前请核对信息 | 请确认学生信息、成绩和志愿方案已复核。 |
| 离开表单 | 当前信息尚未保存 | 离开页面可能导致已填写内容丢失。 |

### 13.3 消息提示类型

| 类型 | 使用场景 |
|---|---|
| success | 保存成功、生成成功、导出成功 |
| warning | 信息不完整、风险较高、需人工复核 |
| error | 保存失败、网络错误、表单错误 |
| info | 普通说明、操作提醒 |

## 16. 页面级规范

## 16.1 登录页

| 项目 | 规范 |
|---|---|
| 背景 | 浅灰或浅蓝渐变背景 |
| 登录卡片宽度 | 380px-420px |
| 标题 | 系统名称，24px，600 |
| 副标题 | 13px-14px 灰色说明 |
| 登录按钮 | 100% 宽度，40px 高 |
| 免责声明 | 放在登录卡片底部，12px 灰色 |

## 16.2 工作台

页面结构：

```text
页面标题区
数据概览卡片区
快捷操作区
最近学生列表
```

规范：

1. 数据概览卡片建议 4 列布局。
2. 高风险学生卡片可使用红色弱化背景。
3. 最近学生列表最多展示 5-8 条。
4. 快捷按钮不超过 4 个。

## 16.3 学生管理页

规范：

1. 筛选区单独使用白色卡片。
2. 表格字段保持横向清晰，不要出现拥挤。
3. 状态列统一使用标签。
4. 操作列固定在右侧，方便用户操作。

## 16.4 学生信息录入页

规范：

1. 顶部使用步骤条。
2. 每一步使用表单卡片承载。
3. 底部按钮固定在卡片底部。
4. 确认提交页以只读摘要形式展示。

## 16.5 估分分析页

规范：

1. 顶部展示学生信息摘要。
2. 核心分数使用大数字突出。
3. 冲稳保使用三张并列卡片。
4. 风险提示放在结果区域下方。

## 16.6 专业推荐页

规范：

1. 推荐专业使用卡片展示。
2. 适配度分数放在卡片右上角。
3. 风险提示放在卡片底部。
4. 不推荐专业单独成区，避免与推荐专业混淆。

## 16.7 志愿方案页

规范：

1. 页面采用三栏布局：冲一冲、稳一稳、保一保。
2. 每栏顶部显示风险说明。
3. 院校卡片内显示分数、位次、城市、推荐理由。
4. 底部必须显示“需以官方数据为准”的提示。

## 16.8 报告生成页

规范：

1. 左侧显示报告目录，右侧显示报告预览。
2. 目录固定，方便跳转。
3. 报告预览区模拟 A4 纸张效果。
4. 导出按钮放在页面右上角或底部固定操作区。
5. 报告中风险声明必须可见。

## 17. 数据可视化规范

### 17.1 图表使用场景

| 图表 | 使用场景 |
|---|---|
| 柱状图 | 各科成绩对比 |
| 折线图 | 模考成绩变化趋势 |
| 雷达图 | 学生能力画像 |
| 进度条 | 专业适配度 |
| 环形图 | 志愿冲稳保比例 |

### 17.2 图表颜色

1. 图表主色使用系统蓝 `#2563EB`。
2. 安全类数据使用绿色。
3. 风险类数据使用橙色或红色弱化。
4. 同一图表内颜色不超过 5 种。

### 17.3 图表使用原则

1. 图表要服务于解释，不为装饰而使用。
2. 图表旁边应有简短文字解释。
3. 坐标轴、图例、单位必须清楚。
4. 高考志愿数据存在不确定性时，不应使用过于绝对的图表表达。

## 18. 空状态、加载状态与异常状态

### 18.1 空状态

| 页面 | 文案 |
|---|---|
| 学生管理 | 暂无学生数据，请点击“新增学生”开始录入。 |
| 报告管理 | 暂无报告，请先选择学生生成规划报告。 |

| 志愿方案 | 当前缺少分数或位次信息，暂无法生成志愿方案。 |

常用文案：

```text
正在加载学生数据...
正在生成估分分析...

```

加载时间较长时，应说明原因：

```text
报告内容较多，生成过程可能需要一点时间，请勿重复点击。
```

### 18.3 异常状态

| 场景 | 文案 |
|---|---|
| 网络错误 | 网络连接异常，请稍后重试。 |
| 接口错误 | 数据获取失败，请联系管理员。 |
| 权限不足 | 当前账号暂无权限访问该页面。 |
| 数据缺失 | 当前学生资料不完整，无法生成完整分析。 |

## 19. 响应式规范

第一阶段以 PC Web 后台为主，建议优先适配 1366px 及以上屏幕。

### 19.1 断点建议

| 屏幕宽度 | 处理方式 |
|---|---|
| ≥1440px | 标准后台布局，内容区可适当加宽 |
| 1200px-1439px | 标准布局，卡片 3-4 列 |
| 1024px-1199px | 左侧菜单可收起，卡片 2-3 列 |
| ＜1024px | 第一阶段可提示使用电脑端访问 |

### 19.2 响应式原则

1. 学生管理、报告生成等页面优先保证 PC 端体验。
2. 小屏下表格可横向滚动。
3. 三栏志愿方案在窄屏下可改为纵向排列。
4. 表单在窄屏下由 2 列改为 1 列。

## 22. 文案风格规范

### 22.1 文案语气

系统文案应理性、稳妥、专业，不制造焦虑，也不做绝对承诺。

### 22.2 推荐表达

| 场景 | 推荐文案 |
|---|---|
| 风险说明 | 当前方案存在一定风险，建议结合正式位次进一步复核。 |
| 报告声明 | 本报告仅作为志愿规划辅助建议，不构成录取承诺。 |
| 信息缺失 | 当前学生资料不完整，请补充后再生成分析。 |
| 成功提示 | 学生信息已保存。 |
| 生成提示 | 报告已生成，建议人工复核后再交付家长。 |

### 22.3 禁止表达

| 不推荐文案 | 原因 |
|---|---|
| 保证录取 | 绝对化承诺，风险极高 |
| 必上本科 | 不符合志愿填报不确定性 |
| 这个专业一定适合你 | 过度确定 |
| 低分捡漏神器 | 娱乐化、营销感过强 |
| 命中率100% | 不可信且容易引发纠纷 |

## 23. 页面验收标准

前端页面完成后，应按照以下标准检查：

| 检查项 | 标准 |

| 操作安全 | 删除、导出、生成报告等操作有确认提示 |

## 25. 总结

本 UI 设计规范用于保证高考志愿规划系统在不同页面、不同开发成员之间保持一致的视觉和交互风格。第一阶段应优先统一颜色、字体、间距、按钮、卡片、表格、标签和表单样式，确保系统具备教育咨询类产品应有的专业感、清晰感和可信感。

前端开发过程中，凡是涉及“学生状态、风险等级、报告生成、志愿方案”的界面，都必须遵循本规范，尤其要避免绝对化承诺和过度营销化表达。系统的最终目标不是制造焦虑，而是帮助咨询师和家长更清楚地理解学生情况，并形成更稳妥的报考规划。
