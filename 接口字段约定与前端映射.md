# 高考志愿规划系统接口字段约定与前端映射

本文档基于当前前端页面结构、`src/api/planning.js` 服务层和 `src/api/contracts.js` 映射规则整理，供前后端联调直接使用。

## 1. 基础约定

### 1.1 接口前缀

- 开发环境建议前缀：`/api`
- 本地 `.env` 示例：`VITE_API_BASE_URL=http://localhost:8080/api`

### 1.2 统一响应包

除文件流下载外，所有接口建议统一返回以下结构：

```json
{
  "code": 0,
  "message": "success",
  "requestId": "req_20260508_0001",
  "timestamp": "2026-05-08T10:30:00+08:00",
  "data": {}
}
```

字段约定：

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | `number` | 业务状态码，`0` 表示成功 |
| `message` | `string` | 提示信息 |
| `requestId` | `string` | 请求链路 ID |
| `timestamp` | `string` | 服务端响应时间，ISO 8601 |
| `data` | `object/array` | 业务数据 |

### 1.3 状态字典建议

#### 学生进度状态 `progressStatus`

| 值 | 文案 |
|---|---|
| `PENDING` | 未填写资料 |
| `BASIC_DONE` | 已完成基础信息 |
| `ANALYSIS_DONE` | 已完成估分 |
| `REPORT_DONE` | 已生成报告 |
| `REVIEWED` | 已人工复核 |
| `DELIVERED` | 已交付家长 |

#### 风险等级 `riskLevel`

| 值 | 文案 |
|---|---|
| `HIGH` | 高风险 |
| `MEDIUM` | 中风险 |
| `LOW` | 低风险 |
| `REVIEW` | 待人工复核 |

#### 志愿分层 `bucketType`

| 值 | 文案 |
|---|---|
| `RUSH` | 冲一冲 |
| `STEADY` | 稳一稳 |
| `SAFE` | 保一保 |

### 2.3 学生列表

#### `GET /students`

用途：获取学生管理列表。

请求参数映射：

| 前端参数 | 后端字段 | 类型 | 说明 |
|---|---|---|---|
| `keyword` | `keyword` | `string` | 姓名或关键词 |
| `province` / `provinceCode` | `provinceCode` | `string` | 省份编码，建议后端统一编码 |
| `status` | `progressStatus` | `string` | 学生进度状态 |
| `examYear` | `examYear` | `number` | 高考年份 |
| `subjects` | `subjectCombination` | `string` | 选科组合 |
| `riskLevel` | `riskLevel` | `string` | 风险等级 |
| `date` / `dateRange[0]` | `updatedStartDate` | `string` | 更新时间起始，`YYYY-MM-DD` |
| `date` / `dateRange[1]` | `updatedEndDate` | `string` | 更新时间结束，`YYYY-MM-DD` |
| `page` | `pageNo` | `number` | 页码，从 1 开始 |
| `pageSize` | `pageSize` | `number` | 每页条数 |
| `orderBy` | `orderBy` | `string` | 排序字段 |
| `orderDirection` | `orderDirection` | `string` | `ASC` / `DESC` |

### 2.7 更新学生档案

#### `PUT /students/{studentId}`

用途：编辑已有学生档案。

前端表单到后端请求体映射：

| 前端字段 | 后端字段 | 类型 | 说明 |
|---|---|---|---|
| `name` | `studentName` | `string` | 学生姓名 |
| `province` | `gaokaoProvince` | `string` | 高考省份 |
| `year` | `examYear` | `number` | 高考年份 |
| `subjects` | `subjectCombination` | `string` | 选科组合 |
| `chinese` | `scores.chinese` | `number` | 语文成绩 |
| `math` | `scores.math` | `number` | 数学成绩 |
| `english` | `scores.english` | `number` | 英语成绩 |
| `physics` | `scores.physics` | `number` | 物理成绩 |
| `chemistry` | `scores.chemistry` | `number` | 化学成绩 |
| `biology` | `scores.biology` | `number` | 生物成绩 |
| `cities` | `preference.targetCities` | `string[]` | 前端逗号分隔字符串会拆成数组 |
| `budget` | `preference.familyBudgetRange` | `string` | 家庭预算 |
| `majorPreference` | `preference.majorPreference` | `string` | 专业偏好说明 |
| `logic` | `profile.logicScore` | `number` | 逻辑能力评分 |
| `expression` | `profile.expressionScore` | `number` | 表达意愿评分 |
| `stability` | `profile.stabilityScore` | `number` | 稳定性评分 |
| `graduate === "可以接受"` | `profile.acceptGraduateExam` | `boolean` | 是否接受考研 |

## 3. 当前前端代码中的参数映射位置

当前映射规则已经沉到以下文件：

| 文件 | 作用 |
|---|---|
| `src/api/contracts.js` | 接口路径、方法、参数转换函数 |
| `src/api/client.js` | 统一 query/body/响应包处理 |
| `src/api/planning.js` | 面向页面的 API 服务封装 |

重点函数：

| 函数 | 作用 |
|---|---|
| `mapStudentListQuery` | 学生列表筛选参数映射 |
| `mapStudentIntakePayload` | 学生录入表单转后端请求体 |
| `mapMajorQuery` | 专业推荐查询参数映射 |
| `mapPlanQuery` | 志愿方案查询参数映射 |
| `mapReportExportBody` | 报告导出请求体映射 |
| `createPath` | 动态路径参数替换，如 `:studentId` |

## 4. 联调建议

1. 后端先按本文档实现统一响应包，前端当前会默认优先读取 `data` 字段。
2. 学生状态、风险等级、志愿分层建议后端返回“稳定枚举值 + 前端展示文案”，不要只返回纯中文文案。
3. 学生列表的 `provinceCode`、`progressStatus`、`riskLevel` 建议后端统一使用枚举，避免前端筛选值漂移。
4. 报告导出如果后端直接返回文件流，需要单独约定 `blob` 下载模式；当前服务层先按 `downloadUrl` 形式约定。
