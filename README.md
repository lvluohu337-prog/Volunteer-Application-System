# 歪歪志愿馆 · 高考志愿智能决策辅助系统

> 一个面向高考生和家长的志愿填报辅助决策系统。核心价值是**帮助没有资源背景的学生，依靠自身天赋（八字/五行/性格分析）+ 真实招生数据，科学地选择适合自己的院校和专业。**

---

## 1. 项目概述

### 项目背景

每年全国有上千万高考生面临志愿填报，但绝大多数学生和家长缺乏系统性的决策支持：

- 热门专业盲目跟风，不考虑学生是否适合
- 信息不对称，普通家庭难以获取科学的择校择专业建议
- 市面上多数工具只查分数线，没有"天赋×数据"的综合决策能力
- 有资源的学生靠咨询师，没资源的学生只能靠运气

### 项目目标

- [x] 通过八字/前六字/五行分析，发现学生天赋方向和发展潜能
- [ ] 建立"天赋→专业大类→具体院校专业"的完整推荐链路
- [ ] 基于真实招生数据（分数、位次、选科、招生计划）进行候选匹配
- [ ] 生成冲稳保分层志愿方案和第一志愿建议
- [ ] 输出可交付的商业化报告（99 元基础版 / 399 元深度版）

### 适用用户

- **高考生**：想了解自己适合什么专业方向的学生
- **家长**：想帮孩子科学决策、但缺少行业经验的家长
- **志愿填报咨询师**：需要数据工具辅助出具方案的专业人士

### 项目边界

- **本项目会做**：学生信息录入、八字/五行/星座画像分析、专业方向推荐、院校专业候选匹配、冲稳保方案生成、报告展示与导出（PDF/Word）
- **本项目暂不做**：在线支付闭环、用户注册/登录/鉴权系统、多角色权限管理、AI 对话交互
- **后续可能扩展**：多方案对比、考研路径分析、就业迁移路径分析、699/999 深度报告

## 8. 核心业务逻辑说明

### 8.1 推荐流程（数据流）

```
学生信息录入
    │
    ▼
八字/五行/星座分析 ← 画像辅助层（解释用，不作为录取依据）
    │
    ▼
天赋方向 → 专业大类映射 ← 核心差异化
    │
    ▼
根据分数/位次/选科筛选可报考院校专业 ← 硬约束层
    │
    ▼
冲/稳/保分层 + 第一志愿推荐 ← 策略层
    │
    ▼
报告生成 + 导出交付 ← 交付层
```

### 8.2 冲稳保分层逻辑

基于学生位次与历史最低录取位次的差距（margin）进行分层：

| 条件 | 分档 |
|------|------|
| margin ≥ 16% 或 margin ≥ 5000 名 | 🟢 保底 |
| margin ≥ 5% 或 margin ≥ 1200 名 | 🔵 稳妥 |
| margin ≥ -8% 或 margin ≥ -1800 名 | 🟡 冲刺 |
| 低于以上条件 | 🔴 不建议（out） |

### 8.3 合规边界

- 八字/五行/星座分析**只能作为专业方向选择与沟通解释的辅助工具**
- 录取判断**必须以分数、位次、选科、招生计划、历史录取数据为准**
- 所有报告必须包含合规声明：**不保证录取结果**
- 禁止出现"包录取""必中""100%准确"等承诺性表述

## 9. 已知问题

- [ ] **冲稳保分层不稳定**：真实场景下所有候选可能全部落入同一档位（如全部为 "冲刺"），需要优化分层阈值和兜底策略
- [ ] **导出为段落型，非表格型**：PDF/Word 导出当前基于 `sections.body` 段落渲染，尚未把结构化推荐表（院校+专业+代码+分数+位次）以表格形式导出
- [ ] **省份支持不均**：仅有河南省有完整的真实招生数据，其他省份数据尚未导入或未达到同等水平
- [ ] **天赋→专业映射偏弱**：当前五行分析更多输出"解释层"结果，尚未形成强映射规则（如"木→教育/文化传播/设计"→具体专业列表）
- [ ] **关键词匹配失效**：候选匹配中 `keyword_hits` 和 `preference_score` 当前固定为 0，偏好匹配实际上未生效
- [ ] **省批次线数据为空**：`province_batches` 表已建但无数据，影响批次判断
- [ ] **综合评分未包含专业适配度和城市适配度**：当前 composite_score 只考虑选科/位次/分数/概率，未包含专业方向和城市产业匹配因素
- [ ] **测试覆盖率不足**：缺乏自动化测试，尤其是推荐引擎回归测试和导出稳定性测试
- [ ] **没有用户鉴权**：系统当前无登录注册，多人使用时缺少数据隔离
